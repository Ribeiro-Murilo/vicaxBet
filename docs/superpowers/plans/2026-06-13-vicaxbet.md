# vicaxBet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a satirical no-money World Cup betting game (monorepo) with login, score predictions, two-currency economy, two separate rankings, and an admin who registers games and posts final scores.

**Architecture:** Monorepo with npm workspaces. `server/` = Express + mysql2 REST API with JWT auth. `web/` = React + Vite SPA. `db/` = MySQL schema + seed. Pure score-scoring logic isolated in a testable module.

**Tech Stack:** Node.js, Express, mysql2, bcryptjs, jsonwebtoken, React, Vite, MySQL.

## API Contract (used by both server and web)

- `POST /api/register {username,password}` -> `{token,user}`
- `POST /api/login {username,password}` -> `{token,user}`
- `GET  /api/me` -> `{user}` (auth)
- `GET  /api/games` -> `[{id,time_a,time_b,data_jogo,gol_a,gol_b,status}]`
- `GET  /api/palpites` (auth) -> user's palpites
- `POST /api/palpites {game_id,palpite_gol_a,palpite_gol_b}` (auth) -> upsert official palpite
- `GET  /api/apostas` (auth) -> user's apostas
- `POST /api/apostas {game_id,palpite_gol_a,palpite_gol_b,valor}` (auth) -> generates odd, debits fichas
- `POST /api/conversao {tipo:"pp_para_fa"|"fa_para_pp", quantidade}` (auth)
- `GET  /api/ranking/principal` -> users ordered by pontos_principais
- `GET  /api/ranking/fichas` -> users ordered by fichas
- `POST /api/admin/games {time_a,time_b,data_jogo}` (admin)
- `POST /api/admin/games/:id/resultado {gol_a,gol_b}` (admin) -> resolves game

User object: `{id,username,is_admin,pontos_principais,fichas}`.

## Task 2: Database schema + seed

**Files:**
- Create: `db/schema.sql`, `db/seed.sql`

- [ ] **Step 1: schema.sql**

```sql
CREATE DATABASE IF NOT EXISTS vicaxbet;
USE vicaxbet;
DROP TABLE IF EXISTS apostas, palpites, games, users;

CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  is_admin TINYINT(1) DEFAULT 0,
  pontos_principais INT DEFAULT 0,
  fichas INT DEFAULT 1000,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE games (
  id INT AUTO_INCREMENT PRIMARY KEY,
  time_a VARCHAR(60) NOT NULL,
  time_b VARCHAR(60) NOT NULL,
  data_jogo DATETIME,
  gol_a INT NULL,
  gol_b INT NULL,
  status ENUM('aberto','resolvido') DEFAULT 'aberto',
  created_by INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE palpites (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  game_id INT NOT NULL,
  palpite_gol_a INT NOT NULL,
  palpite_gol_b INT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_user_game (user_id, game_id)
);

CREATE TABLE apostas (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  game_id INT NOT NULL,
  palpite_gol_a INT NOT NULL,
  palpite_gol_b INT NOT NULL,
  valor INT NOT NULL,
  odd DECIMAL(5,2) NOT NULL,
  status ENUM('pendente','ganhou','perdeu') DEFAULT 'pendente',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
